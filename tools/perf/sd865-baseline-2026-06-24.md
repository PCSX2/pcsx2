# SD865 CPU bottleneck baseline — 2026-06-24

First real Snapdragon 865 profile of the ARM64 port (Retroid Flip 2 / ROCKNIX).
Supersedes the stale RK3562 numbers AND the M2-Asahi baseline as the *gating* data
for neither/LRPS2 cherry-pick relevance decisions. M2 was only methodology validation;
this is the deploy target.

## Setup (reproduce)

- Device: `root@192.168.4.44` — ROCKNIX nightly, SM8250, kernel 7.0.x, glibc 2.41,
  **4K pages**, Adreno 650 Vulkan. 8 cores: **0-3 = A55 (part 0x805), 4-7 = A77 (0xd0d)**.
- Self-contained tree at `/storage/pcsx2` (`/root` is RO squashfs; root's HOME is `/storage`).
  Binary built on M2 from the `clang-sd865` preset (`CMakeUserPresets.json`:
  build-perf clone with `HOST_PAGE_SIZE=0x1000`, `HOST_CACHE_LINE_SIZE=64`, `USE_BACKTRACE=OFF`)
  + its full dynamic-lib closure staged to `/storage/pcsx2/lib` (incl. dlopen'd
  `libshaderc_shared`/`libSPIRV-Tools*`, which ldd misses). The M2 16K-page binary aborts
  here on the page-size hardware check — the 4K rebuild is mandatory.
- ROMs already on the device SD card at `/storage/roms/ps2`; savestates + BIOS under
  `/storage/pcsx2/.config/PCSX2`.
- Run: `tools/perf/profile_run.sh --device sd865 --scene <s> --renderer null --runs 3`
  (sd865.env sets HOME/LD_LIBRARY_PATH, `PIN="taskset -c 4-7"` to force the A77 cores,
  `cycles` event). `perf_event_paranoid=1` first. Pipeline cache must be warm (first cold
  run compiles Adreno pipelines on the GS thread — async MTGS keeps it off the EE).

renderer=null = CPU-emulation shape; eerunner still spins a *surfaceless Adreno Vulkan
device* (no draws), so the `GS` bucket is GIF/register CPU work only. Audio OUTPUT
excluded (SPU2 Backend=Null) but the SPU2 DSP core still runs and is counted.

## Read-out for the neither funnel relevance gate

- **EE-JIT is the single biggest consumer** (15-25% across both games) → EE-side neither
  picks (N-03 flat block LUT dispatch; N-01 if it covers EE block linking) are HIGH relevance.
- **VU is enormous in UYA**: VU1 12.89 + VU0 9.60 + VU-glue 13.42 = **35.9%**. SotC VU1+glue ~18%.
  - **N-01 (VU block linking)** → HIGH: VU-glue (lookup/dispatch envelope) is a *measured*
    13.4% in UYA — exactly what block linking attacks.
  - **N-02 (COP2Native / VU0)** → **WARM, and this FLIPS the M2 read**: M2 showed VU0≈0 so I
    had it as "park"; UYA on SD865 shows **VU0-JIT 9.60%** (COP2 macro mode). The SD865 profile
    is why we gate on the device, not M2.
- **MTVU/MTGS sync 11% (SotC)** — async ring-buffer coordination is real; threading-model
  deltas could matter.
- **VIF 6-9%** → N-07 VIF idioms MEDIUM.
- GS 11-17% even under null (GIF/register CPU) — not directly on the N-list.

Updated Class-1 priority by *measured* SD865 relevance: **N-01 ≈ N-03 > N-02 > N-07**.
(Was N-01 > N-03 > [N-02 parked] on the M2 guess.)
